cassandra_rest.viewsets.register_viewset(app, viewset, model, fields=None, pks=None, endpoint=None, url=None, preprocessors=None, postprocessors=None, ignore_pks_on_update=True)[source]¶

Sets up the viewset to respond to url endpoints for the specified model

Parameters:
  • app (flask.Flask) – The flask app to use
  • viewset (flask.views.View) – The cassandra_rest viewset to use
  • model (cqlengine.models.Model) – The cqlengine model to use
  • fields (list) – The fields on the model to use. By default it will grab all of the fields on the specified model
  • pks (list) – The primary keys on which the API will respond to GET, PUT, and DELETE operations. The url for these operations will be <url>/<pk1>/<pk2>/.../<pkn>
  • endpoint (str) – The name of the endpoint. Defaults to <model>_api
  • url (str) – The base url on which the api for this model responds
Returns:

None
